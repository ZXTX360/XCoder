🤝 Contributing to XCoder
First off, thank you for considering contributing to XCoder! It’s people like you who make XCoder a powerful tool for everyone.
📜 Code of Conduct
By participating in this project, you agree to abide by our Code of Conduct. Please be respectful and constructive in all interactions.
🛠 How Can I Contribute?
1. Reporting Bugs

    Check the Issues tab to see if the bug has already been reported.
    If not, open a new issue using the "Bug Report" template.
    Include steps to reproduce, expected behavior, and screenshots if applicable.

2. Suggesting Enhancements

    We love new /slash command ideas!
    Open an issue labeled enhancement to discuss your idea before starting work.

3. Pull Requests (PRs)

    Fork the repo and create your branch from main.
    Install dependencies via npm install or pip install -r requirements.txt.
    Write your code. If you use AI (like Copilot or XCoder itself) to generate parts of the PR, please disclose it in the description.
    Ensure tests pass. Run npm test or the equivalent test suite.
    Submit the PR with a clear description of the changes.

🤖 AI Contribution Policy
As an AI-centric project, we encourage using AI to help you code. However:

    Human Oversight: You are responsible for the code you submit. Ensure you understand it.
    No Hallucinations: Double-check that AI-generated documentation or comments are accurate.

🎨 Style Guide

    Linting: We use Prettier/ESLint (or Black for Python). Please run the linter before committing.
    Commit Messages: Use conventional commits (e.g., feat: add /ship command, fix: resolve logic in context engine).
